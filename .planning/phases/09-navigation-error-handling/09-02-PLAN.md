# Plan 09-02: Audit Mobile Navigation

## Objective
Verify all mobile routes, tabs, and navigation work correctly

## Scope
Audit only - Document all gaps, defer fixes to Phase 10

---

## Files to Read

- `apps/mobile/src/app/(tabs)/` - tab routes
- `apps/mobile/src/app/` - stack routes and _layout files
- `apps/mobile/src/app/_layout.jsx` - root layout
- Components with router.push or Link usage

---

## Tasks

### Task 1: Audit Tab Bar Navigation

**Actions:**
1. Find the tabs layout file `apps/mobile/src/app/(tabs)/_layout.jsx`
2. List all tab routes defined
3. Verify tab icons and labels are present
4. Check that each tab file exists

**Expected output:**
- Tab configuration table
- Any missing tab screens

---

### Task 2: Audit Stack Navigation

**Actions:**
1. List all route files outside (tabs) folder
2. Map detail screens (clients, incidents, employees, etc.)
3. Check for proper back navigation patterns
4. Document deep link structure

**Expected output:**
- Stack route map
- Navigation hierarchy

---

### Task 3: Audit router.push/navigate Calls

**Actions:**
1. Search for `router.push`, `router.navigate`, `router.replace`
2. Search for `<Link>` components with `href`
3. For each navigation call, verify target route exists
4. Document missing routes (e.g., `/clients/new`)

**Expected findings:**
- Missing `/clients/new` route (referenced in clients/index.jsx:83)

---

### Task 4: Document Navigation Gaps

**Actions:**
1. Compile all missing screens
2. Identify navigation dead-ends
3. Create recommendations for Phase 10

**Deliverables:**
- Gap analysis table
- Priority recommendations

---

## Success Criteria

- [ ] All tab routes audited
- [ ] Stack navigation mapped
- [ ] router.push calls verified
- [ ] Navigation gaps documented
- [ ] 09-02-SUMMARY.md created

---

## Output Format

Create `09-02-SUMMARY.md` with:
1. Tab route table
2. Stack route map
3. Navigation calls verification table
4. Missing routes list
5. Phase 10 recommendations
